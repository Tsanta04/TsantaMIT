Netfilter

It allows administrators to configure rules and filters to control network traffic entering and exiting a system.

New rules

Allow specific incoming, outgoing, forwarding connections, allow established and related connections, drop or reject unwanted traffic, and limit the number of connections.

Default rules

Determine the policy for packets that do not match any of the explicitly defined rules. There are three main default policies: ACCEPT, DROP, and REJECT.

Current rules

List the currently configured firewall rules, displays the rules for each of the built-in chains in the filter table (the default table for packet filtering) of the iptables firewall.

Why do we need to filter the connection?

Filtering connections allows administrators to control which network traffic is allowed to enter or leave a system or network. By defining rules based on criteria such as source IP address, destination IP address, port numbers, and protocols, administrators can enforce access control policies to permit or deny specific types of connections.

Filtering connections helps enhance network security by blocking unauthorized access attempts, malicious traffic, and potential security threats. By blocking unwanted connections, iptables can protect against various network-based attacks, including port scanning, denial-of-service (DoS) attacks, and intrusion attempts.

Firewall functionaity

It can be used to set up a firewall on Linux systems, allowing administrators to filter incoming and outgoing traffic based on various criteria such as IP addresses, ports, protocols, and packet states.
